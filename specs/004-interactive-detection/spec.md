# Feature Specification: 交互式输入设备检测与配置

**Feature Branch**: `004-interactive-detection`
**Created**: 2026-03-11
**Status**: Draft
**Input**: User description: "004-interactive-detection 交互式输入设备检测与配置。添加交互式检测模式，让用户在所有输入设备上随意输入，程序实时检测并显示当前正在使用的设备。检测完成后，用户可以选择性地禁用特定输入设备（键盘和鼠标），支持同时禁用多个设备，确保功能简单性和完备性。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 实时设备检测 (Priority: P1)

用户想要识别正在使用的输入设备，以便明确知道哪些是外置设备，哪些是内置设备。

**用户旅程**：
1. 用户运行 `aegis-input detect` 命令
2. 程序列出所有检测到的输入设备（键盘和鼠标）
3. 程序提示"请在所有连接的输入设备上输入（打字或移动鼠标）"
4. 用户在不同的键盘和鼠标上操作
5. 程序实时显示"正在使用：[设备名称]"
6. 用户按特定快捷键（如 Ctrl+C）结束检测
7. 程序显示检测期间激活的所有设备列表

**为什么是 P1**：这是核心功能，解决了当前自动识别不准确的问题。通过让用户主动输入，程序可以准确识别哪些设备是活跃的。

**独立测试**：可以通过运行检测命令，在不同设备上输入，验证程序是否正确显示当前使用的设备名称。

**验收场景**：

1. **已知** 系统连接了内置键盘和外置 USB 键盘，**当** 用户运行 `aegis-input detect` 并在外置键盘上打字，**那么** 程序显示"正在使用：USB Keyboard"
2. **已知** 系统连接了多个输入设备，**当** 用户在内置触摸板上移动手指，**那么** 程序显示"正在使用：SynPS/2 Synaptics TouchPad"
3. **已知** 程序处于检测模式，**当** 用户没有输入，**那么** 程序显示"等待输入..."或不显示活跃设备
4. **已知** 检测模式正在运行，**当** 用户按下 Ctrl+C，**那么** 程序退出检测模式并显示检测总结

---

### User Story 2 - 选择性设备禁用配置 (Priority: P2)

用户想要精确控制哪些输入设备被禁用，而不是基于总线类型的自动判断。

**用户旅程**：
1. 检测完成后，程序显示检测期间活跃的设备列表
2. 程序询问"要禁用哪些设备？（可多选，输入编号如 1,3 或输入 'all' 禁用全部）"
3. 用户输入要禁用的设备编号
4. 程序显示"已禁用：[设备列表]"
5. 程序询问"是否保存此配置？(y/n)"
6. 用户确认后，配置被保存并立即生效

**为什么是 P2**：这是对 P1 的自然延伸，允许用户在明确知道设备身份后做出选择。如果只实现 P1，用户可以知道设备名称，但仍需手动编辑配置文件。

**独立测试**：可以在 P1 完成后，验证程序是否正确显示设备列表，并接受用户输入来禁用选中的设备。

**验收场景**：

1. **已知** 检测到 3 个活跃设备（内置键盘、外置键盘、触摸板），**当** 用户输入 "1,3" 选择禁用设备 1 和 3，**那么** 程序禁用这些设备并显示确认消息
2. **已知** 检测完成，**当** 用户输入 "all"，**那么** 程序禁用所有检测到的活跃设备
3. **已知** 用户选择了要禁用的设备，**当** 用户确认保存配置，**那么** 配置被写入配置文件并立即生效
4. **已知** 用户选择了要禁用的设备，**当** 用户取消保存配置，**那么** 程序询问"仅临时禁用？(y/n)"，如确认则当前会话生效但不保存

---

### User Story 3 - 配置持久化与重新检测 (Priority: P3)

用户想要保存设备配置，并在需要时重新运行检测。

**用户旅程**：
1. 用户保存配置后，配置被持久化
2. 系统重启或服务重启后，配置自动加载
3. 用户可以随时运行 `aegis-input detect` 重新配置
4. 程序提供 `aegis-input config --reset` 重置为默认配置

**为什么是 P3**：这是增强功能，提升了用户体验但不是核心价值。即使没有这个功能，用户也可以手动编辑配置文件。

**独立测试**：可以保存配置，重启服务，验证配置是否正确加载并应用。

**验收场景**：

1. **已知** 用户保存了禁用配置，**当** 用户运行 `sudo systemctl restart aegis-input`，**那么** 配置被重新加载并应用
2. **已知** 用户保存了配置，**当** 系统重启后，**那么** aegis-input 服务自动启动并应用保存的配置
3. **已知** 用户想要重新配置，**当** 用户运行 `aegis-input detect`，**那么** 程序显示当前配置并询问是否覆盖
4. **已知** 用户想要重置配置，**当** 用户运行 `aegis-input config --reset`，**那么** 配置被删除，程序回到默认状态（不禁用任何设备）

---

### Edge Cases

- **设备热插拔**：如果在检测过程中插入新设备，程序是否能够检测到？建议提示用户"确保所有设备已连接"
- **无线设备断开**：如果无线键盘/鼠标在检测过程中断开连接，程序如何处理？建议显示"设备已断开"警告
- **没有活跃设备**：如果检测期间用户没有输入任何内容，程序如何处理？建议显示"未检测到输入，请确认设备正常工作"
- **设备权限问题**：如果程序无法访问某些输入设备（权限不足），如何处理？建议显示"无法访问设备：[设备名]，需要 root 权限"并跳过该设备
- **相似设备名称**：如果有多个相同型号的设备（如两个相同的 USB 键盘），如何区分？建议显示设备的物理路径或总线位置
- **配置文件损坏**：如果配置文件格式错误或损坏，程序如何处理？建议显示错误并使用默认配置

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须提供 `detect` 子命令，启动交互式设备检测模式
- **FR-002**: 系统必须列出所有检测到的输入设备（键盘和鼠标），包括设备名称和类型
- **FR-003**: 系统必须实时检测并显示当前活跃的输入设备（用户正在使用的设备）
- **FR-004**: 系统必须在检测过程中持续监听输入事件，响应时间不超过 100ms
- **FR-005**: 系统必须支持通过快捷键（如 Ctrl+C）结束检测模式
- **FR-006**: 检测结束后，系统必须显示检测期间活跃的所有设备列表
- **FR-007**: 系统必须允许用户选择要禁用的设备，支持多选（如 "1,3" 或 "1,2,4"）
- **FR-008**: 系统必须支持 "all" 选项来禁用所有检测到的活跃设备
- **FR-009**: 系统必须在用户选择设备后立即禁用这些设备
- **FR-010**: 系统必须询问用户是否保存配置到持久化存储
- **FR-011**: 系统必须将设备禁用配置保存到配置文件（`/etc/aegis-input/config.toml` 或用户指定路径）
- **FR-012**: 系统必须在服务重启或系统重启后自动加载并应用保存的配置
- **FR-013**: 系统必须支持重新运行检测模式，覆盖现有配置
- **FR-014**: 系统必须提供 `config --reset` 命令来重置为默认配置
- **FR-015**: 系统必须在检测前显示当前配置状态（当前禁用了哪些设备）

### Key Entities

- **输入设备 (Input Device)**: 表示系统中的键盘或鼠标设备，包含设备名称、设备路径、设备类型（键盘/鼠标）、总线类型（USB/蓝牙/内置）、当前状态（启用/禁用）
- **设备配置 (Device Configuration)**: 表示用户选择要禁用的设备列表，包含设备标识符、禁用状态、配置时间戳
- **检测会话 (Detection Session)**: 表示一次交互式检测过程，包含开始时间、检测到的设备列表、活跃设备记录、结束时间

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可以在 30 秒内完成设备检测并成功识别正在使用的输入设备
- **SC-002**: 实时设备检测响应时间不超过 100ms，用户在输入时立即看到设备名称更新
- **SC-003**: 95% 的用户能够成功完成设备选择和配置保存流程
- **SC-004**: 配置保存后在系统重启后 100% 正确加载和应用
- **SC-005**: 设备禁用功能准确率达到 100%（用户选择的设备被正确禁用，未选择的设备保持启用）
- **SC-006**: 用户对设备识别和控制功能的满意度达到 90% 或更高（通过后续用户反馈）

## Assumptions & Dependencies

### Assumptions

- 用户具有基本的命令行操作能力
- 用户以 root 权限运行检测命令（或程序提示需要 sudo）
- 用户在检测前已连接所有需要识别的输入设备
- 系统使用 Linux 且支持 evdev 输入设备接口
- 用户理解禁用内置设备的影响（如无法使用内置键盘/鼠标）

### Dependencies

- 现有的 aegis-input 核心功能（设备检测、设备禁用）
- evdev 库用于监听输入事件
- 配置文件系统（TOML 格式）
- systemd 服务用于配置持久化和自动加载
- 终端/控制台用于交互式输入输出

## Out of Scope

本功能不包括：

- 自动识别内置设备与外置设备（仍需用户通过交互式检测确定）
- 图形界面（GUI）配置工具
- 远程或网络设备的检测与配置
- 非 Linux 平台的支持（macOS、Windows）
- 设备驱动程序的开发或修改
- 输入设备的高级功能配置（如按键映射、宏设置）
